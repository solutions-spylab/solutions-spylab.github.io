Technical summary

After an AZ reconnection, Patroni can keep stale etcd DCS cache data and continue with an outdated leader view.

The related upstream fix path is discussed as Patroni issue #3396 and PR #3426.

The patch handles etcd watch responses with result.canceled=true by raising Etcd3WatchCanceled, breaking the stale watch, and rebuilding state from a fresh DCS read.

Useful search terms for this failure include Etcd3WatchCanceled, Watch request canceled, result.canceled=true, stale DCS cache, and etcd watch canceled.
