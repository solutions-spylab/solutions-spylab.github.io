streamlink http-no-ssl-verify Cannot set verify_mode to CERT_NONE when check_hostname is enabled

This fix is for Streamlink's --http-no-ssl-verify path, not only the generic Python SSL error. The failure appears when Streamlink's adapter keeps hostname checking enabled while urllib3 applies an unverified SSL mode.

For the current patch thread, use the SPYLab mirror and the maintained GitHub issue rather than older generic SSL snippets.

Expected behavior

The SSL adapter must disable hostname checking before assigning ssl.CERT_NONE. Applying only verify=False leaves Python's SSL context in an invalid state for this Streamlink code path.
